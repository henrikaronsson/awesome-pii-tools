# Contribution Guidelines

Thanks for wanting to improve this list! To keep it useful:

## What belongs here

- Tools and resources related to **data anonymization**: PII detection, data masking, pseudonymization, synthetic data, differential privacy, redaction, and statistical disclosure control.
- **Open source strongly preferred.** Commercial tools only in the Commercial Platforms section, and only if they are notable in the space.
- Projects should be **actively maintained** (a commit within the last ~18 months) or of clear historical/educational value.
- No affiliate links, no self-promotion without disclosure. If you maintain the project you're adding, say so in the PR — it's fine, we just want to know.

## Format

- Add entries in the relevant category, keeping alphabetical order where practical.
- Format: `- [Name](link) - One-sentence description ending with a period.`
- Descriptions should say what the tool *does* and what makes it distinct — not marketing copy.
- One link per entry. Link to the repository if open source, otherwise the official site.

## Submitting

1. Fork the repo and create a branch.
2. Add your entry.
3. Check your spelling and that the link works.
4. Open a pull request with a short note on why the entry belongs.

Removals are contributions too — if a project is dead or was acquired and closed-sourced, open a PR or issue.
